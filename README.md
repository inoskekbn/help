# GamePC Store - Интернет-магазин игровых компьютеров

## Описание проекта

**CYBERPC** — это современный и удобный веб-сайт для продажи готовых игровых компьютеров и отдельных комплектующих. Проект предназначен для геймеров и энтузиастов, которые хотят быстро и легко собрать или приобрести мощный ПК для игр.

**Основные функции:**
*   **Каталог товаров:** Просмотр готовых сборок и комплектующих (процессоры, видеокарты, ОЗУ и т.д.) с фильтрацией и поиском.
*   **Корзина покупок:** Добавление товаров в корзину, изменение количества и оформление заказа.
*   **Система сборки ПК:** Интерактивный конструктор для выбора совместимых комплектующих.
*   **Адаптивный дизайн:** Удобный просмотр на любом устройстве (ПК, планшет, смартфон).

### **1. Введение**

#### **1.1. Назначение проекта**
Проект CYBERPC представляет собой инновационное веб-приложение, разрабатываемое в рамках учебного курса по современным веб-технологиям. Данная платформа призвана кардинально упростить и демократизировать процесс подбора и покупки компьютерных компонентов, сделав его доступным для самой широкой аудитории пользователей. В современном мире, где компьютер стал неотъемлемой частью повседневной жизни, многие пользователи сталкиваются с проблемой сложности самостоятельной сборки ПК из-за необходимости учитывать множество технических нюансов и совместимости компонентов. CYBERPC решает эту проблему, предлагая интеллектуальный конструктор с автоматической проверкой совместимости.

#### **1.2. Область применения**
Изначально проект создается как демонстрационная платформа для образовательных целей, позволяющая на практике освоить полный цикл разработки современного веб-приложения - от проектирования архитектуры и пользовательского интерфейса до реализации сложной бизнес-логики и развертывания готового решения. Однако, благодаря продуманной архитектуре и расширяемости, в перспективе система может быть доработана до полноценного коммерческого продукта, способного конкурировать на рынке онлайн-продаж компьютерной техники. Платforma также может служить основой для более специализированных решений в области конфигурирования сложной техники.

#### **1.3. Целевая аудитория**
Целевая аудитория проекта охватывает широкий спектр пользователей с различным уровнем технической подготовки. Это и студенты, нуждающиеся в недорогих компьютерах для учебы; и геймеры, требующие максимальной производительности для комфортной игры в современные проекты; и IT-специалисты, ищущие надежные рабочие станции; и просто обычные пользователи, желающие обновить домашний компьютер без необходимости глубокого погружения в технические детали. Особое внимание уделяется новичкам, которые ранее не сталкивались со сборкой компьютеров и нуждаются в пошаговом руководстве и гарантии совместимости выбранных компонентов.

### **2. Техническое задание**

#### **2.1. Функциональные требования**

**2.1.1. Модуль каталога товаров**
Система должна предоставлять пользователям возможность удобного просмотра всего ассортимента товаров, организованного в логическую иерархическую структуру категорий и подкатегорий. Необходима реализация интеллектуальной системы поиска, позволяющей находить товары как по названиям, так и по техническим характеристикам. Для каждого товара должна отображаться детальная карточка с полным описанием, техническими спецификациями, качественными изображениями и актуальной ценой. Система рейтингов и отзывов поможет пользователям принимать более обоснованные решения на основе опыта других покупателей.

**2.1.2. Конструктор ПК**
Ключевой модуль системы - интерактивный конструктор, который проводит пользователя через пошаговый процесс сборки компьютера, начиная от выбора корпуса и заканчивая подбором системы охлаждения. На каждом этапе система автоматически проверяет совместимость выбранных компонентов, исключая возможность ошибок. Алгоритмы конструктора должны учитывать сотни параметров совместимости - от физических размеров до электрических характеристик. Пользователи могут сохранять промежуточные результаты сборки, сравнивать разные конфигурации и делиться ими с другими пользователями.

**2.1.3. Система пользователей**
Многоуровневая система управления пользователями предоставляет различные возможности в зависимости от роли. Обычные пользователи могут регистрироваться, создавать личные кабинеты, управлять своими данными, отслеживать историю заказов и создавать списки избранных товаров. Администраторы получают расширенные права для управления каталогом, обработки заказов и аналитики. Система должна обеспечивать безопасное хранение персональных данных и надежную аутентификацию.

**2.1.4. Система заказов**
Полнофункциональная система управления заказами охватывает весь цикл - от добавления товаров в корзину до финальной доставки. Пользователи могут легко оформлять заказы, выбирать способы доставки и оплаты, отслеживать статус выполнения заказа в реальном времени. Система уведомлений информирует пользователей о ключевых этапах обработки заказа. Для администраторов предусмотрен удобный интерфейс управления заказами с возможностью массовых операций.

#### **2.2. Нефункциональные требования**
В условиях современного интернета производительность играет ключевую роль, поэтому время загрузки страниц не должно превышать 3 секунд даже при нестабильном соединении. Адаптивный дизайн обеспечивает корректное отображение и удобство использования на всех типах устройств - от смартфонов до настольных компьютеров. Кроссбраузерная совместимость гарантирует стабильную работу во всех популярных браузерах. Особое внимание уделяется безопасности - система должна защищать персональные данные пользователей и предотвращать распространенные веб-угрозы.

### **3. Архитектура системы**

#### **3.1. Общая структура**
Архитектура проекта построена по классической трехзвенной модели, обеспечивающей четкое разделение ответственности между компонентами. Клиентская часть, реализованная на современном фреймворке React, отвечает за представление данных и взаимодействие с пользователем. Серверная часть на Node.js обрабатывает бизнес-логику и обеспечивает работу API. База данных MongoDB хранит всю необходимую информацию. Такое разделение позволяет независимо масштабировать компоненты системы и упрощает процесс разработки и тестирования.

```
┌─────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│   Frontend      │    │    Backend API   │    │   Database       │
│   (React)       │◄──►│   (Node.js)      │◄──►│   (MongoDB)      │
└─────────────────┘    └──────────────────┘    └──────────────────┘
```

#### **3.2. Frontend архитектура**
**Технологический стек:**
Для реализации клиентской части выбран React 18+ как один из самых популярных и развитых фреймворков с богатой экосистемой. Маршрутизация осуществляется с помощью React Router, обеспечивающего удобную навигацию между разделами приложения. Для взаимодействия с сервером используется Axios - надежная библиотека для выполнения HTTP-запросов. Стилизация компонентов реализована через CSS Modules, что обеспечивает изоляцию стилей и предотвращает конфликты.

**Основные компоненты:**
- `Header` - интеллектуальная шапка сайта с адаптивной навигацией, поиском и корзиной
- `ProductCatalog` - каталог товаров с расширенными возможностями фильтрации и сортировки
- `PCBuilder` - сложный компонент конструктора ПК с пошаговым интерфейсом
- `ShoppingCart` - корзина покупок с расчетом стоимости и условиями доставки
- `UserProfile` - личный кабинет с историей заказов и управлением профилем

#### **3.3. Backend архитектура**
**Технологический стек:**
Серверная часть построена на Node.js с использованием фреймворка Express.js, что обеспечивает высокую производительность и простоту разработки. Для работы с базой данных используется MongoDB вместе с ODM Mongoose, предоставляющим удобный интерфейс для работы с данными. Аутентификация реализована через JWT-токены, обеспечивающие безопасный доступ к защищенным ресурсам. Для отправки email-уведомлений применяется библиотека Nodemailer.

**Основные модули:**
```javascript
// Структура проекта отражает лучшие практики организации кода
/src
  /controllers    // Обработчики запросов, содержащие бизнес-логику
  /models         // Модели данных с валидацией и хуками
  /routes         // Маршруты API с middleware аутентификации
  /middlewares    // Промежуточное ПО для обработки ошибок и логирования
  /utils          // Вспомогательные функции и утилиты
  /services       // Сервисный слой для сложной бизнес-логики
```

#### **3.4. База данных**
**Основные коллекции:**
Структура базы данных спроектирована для эффективного хранения и быстрого доступа к данным. Коллекция `users` содержит информацию о пользователях и их предпочтениях. `products` хранит полные данные о товарах с поддержкой различных характеристик для разных категорий. `categories` организует товары в древовидную структуру. `orders` управляет всей информацией о заказах, а `configurations` сохраняет пользовательские сборки ПК для повторного использования.

#### **4.1. Базовый URL**
Для доступа к API используется единая точка входа, что упрощает клиентскую разработку и обеспечивает согласованность интерфейса. Все запросы должны содержать корректные заголовки аутентификации для защищенных endpoints.
```
http://localhost:3000/api
```

#### **4.2. Основные endpoints**

**4.2.1. Товары**
Товарный API предоставляет полный набор операций для управления каталогом. Поддерживается пагинация, фильтрация по множеству параметров, полнотекстовый поиск и сортировка. Административные endpoints защищены дополнительной проверкой прав доступа.
```javascript
GET    /products           // Получить список товаров с поддержкой фильтрации
GET    /products/:id       // Получить детальную информацию о товаре по ID
```


### **6. Конструктор ПК**

#### **6.1. Алгоритм проверки совместимости**
Алгоритм проверки совместимости представляет собой сложную систему правил и условий, учитывающую технические характеристики, физические размеры и электрические параметры компонентов. Система последовательно проверяет различные аспекты совместимости, предоставляя детальную информацию о найденных проблемах.
```javascript
// Расширенный пример проверки совместимости компонентов
function checkCompatibility(selectedComponents) {
  const issues = [];
  
  // Проверка процессора и материнской платы
  const cpu = selectedComponents.find(c => c.category === 'cpu');
  const motherboard = selectedComponents.find(c => c.category === 'motherboard');
  
  if (cpu && motherboard) {
    if (cpu.specifications.socket !== motherboard.specifications.socket) {
      issues.push({
        type: 'critical',
        message: `Процессор ${cpu.name} не совместим с материнской платой ${motherboard.name} (разные сокеты)`,
        components: [cpu._id, motherboard._id]
      });
    }
    
    if (cpu.specifications.tdp > motherboard.specifications.maxTdp) {
      issues.push({
        type: 'warning',
        message: `TDP процессора превышает максимально поддерживаемый материнской платой`,
        components: [cpu._id, motherboard._id]
      });
    }
  }
  
  // Проверка оперативной памяти
  const ram = selectedComponents.find(c => c.category === 'ram');
  if (ram && motherboard) {
    if (ram.specifications.type !== motherboard.specifications.memoryType) {
      issues.push({
        type: 'critical',
        message: `Тип оперативной памяти не поддерживается материнской платой`,
        components: [ram._id, motherboard._id]
      });
    }
  }
  
  // Проверка физических размеров
  const case = selectedComponents.find(c => c.category === 'case');
  const gpu = selectedComponents.find(c => c.category === 'gpu');
  
  if (case && gpu) {
    if (gpu.specifications.length > case.specifications.maxGPULength) {
      issues.push({
        type: 'critical',
        message: `Видеокарта слишком длинная для выбранного корпуса`,
        components: [gpu._id, case._id]
      });
    }
  }
  
  return {
    isCompatible: issues.filter(i => i.type === 'critical').length === 0,
    issues: issues,
    warnings: issues.filter(i => i.type === 'warning')
  };
}
```

#### **6.2. Порядок сборки**
Процесс сборки компьютера разбит на логические этапы, каждый из которых учитывает зависимости между компонентами. Такой подход гарантирует, что выбор каждого последующего компонента ограничен совместимостью с уже выбранными элементами.
1. **Выбор корпуса** - определяет физические ограничения для других компонентов
2. **Выбор материнской платы** - задает платформу и поддерживаемые стандарты
3. **Выбор процессора** - определяет вычислительную мощность системы
4. **Выбор оперативной памяти** - с учетом типа и максимальной поддерживаемой частоты
5. **Выбор видеокарты** - для графической производительности
6. **Выбор накопителей** - SSD и HDR с учетом доступных портов
7. **Выбор блока питания** - расчет необходимой мощности с запасом
8. **Выбор системы охлаждения** - адекватное охлаждение для выбранных компонентов

Функциональное тестирование охватывает все ключевые сценарии использования приложения. Интеграционные тесты проверяют взаимодействие между различными модулями системы и внешними сервисами. Тестирование API endpoints обеспечивает стабильность программного интерфейса. Тесты взаимодействия с базой данных проверяют корректность выполнения запросов и целостность данных. Отдельное внимание уделяется тестированию системы аутентификации и авторизации.

#### **8.3. Пример тест-кейса**
Детальные тест-кейсы описывают конкретные сценарии использования системы с ожидаемыми результатами. Каждый тест проверяет определенный аспект функциональности и помогает выявить регрессии при внесении изменений в код.
```javascript
describe('PC Builder Compatibility Check', () => {
  it('should detect incompatible CPU and motherboard combination', () => {
    // Подготовка тестовых данных
    const cpu = { 
      name: 'AMD Ryzen 7 5800X', 
      specifications: { socket: 'AM4', tdp: 105 } 
    };
    const motherboard = { 
      name: 'ASUS Prime Z690-P', 
      specifications: { socket: 'LGA1700', maxTdp: 150 } 
    };
    
    // Выполнение тестируемой функции
    const result = checkCompatibility([cpu, motherboard]);
    
    // Проверка ожидаемых результатов
    expect(result.isCompatible).toBe(false);
    expect(result.issues).toHaveLength(1);
    expect(result.issues[0].type).toBe('critical');
  });
  
  it('should allow compatible components and calculate total power', () => {
    const compatibleComponents = [
      { category: 'cpu', specifications: { tdp: 65 } },
      { category: 'gpu', specifications: { powerConsumption: 180 } },
      { category: 'ram', specifications: { powerConsumption: 15 } }
    ];
    
    const result = checkCompatibility(compatibleComponents);
    const totalPower = calculateTotalPower(compatibleComponents);
    
    expect(result.isCompatible).toBe(true);
    expect(totalPower).toBe(260);
    expect(result.issues).toHaveLength(0);
  });
});
```
